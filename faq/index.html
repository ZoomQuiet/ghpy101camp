课程形式¶

网络私墪式实训课程

问: 上课日期我看到了，但是每天上课的时间是几点？我也是个有娃要鸡的老爸[愉快]

答: 定期直播结合自主学习时间

  • 蟒营™课程不同其它任何网络课程, 不以视频/教材为核心
  • 而是以学员真实编程为核心,
  • 具体课程时间点, 正式开课后:
    • 每周日 2042~ 至少一小时直播交流
    • 每周三 2000~2042 定长42分钟答疑
    • 以上为课程赠送服务
  • 而真正的学习时间段, 由学员们自行灵活安排, 确保每周10+小时投入到课程中来
  • 每天任何时刻都可以通过各种渠道和课程以及同学们交流

问:没看明白在线上课是在github上？是怎么用啊？不好意思，因为太小白，不知道怎么用gitlab，不知道怎么参加上课😂😂 , 也在知识星球上，查了半天还是没看懂怎么上课[捂脸][捂脸] 」

答: 蟒营™是原创网络私墪式实训课程

  • 和其它以网络课程最大的不同就在, 完全仿真式开发环境,
  • 就象:
    • 厨师, 在各自案头/灶台, 根据菜谱自主完成菜品, 厨房内用吼声协调进展
    • 建筑工, 在各自楼层, 根据设计图完成责任区段, 工地内部用步话机协调
    • 软件工程师, 在各自工位自己电脑上, 根据需求说明书自主完成开发, 团队内部通过项目管理系统协调
    • 蟒营™课程, 学员在各自本地电脑上, 根据课程任务说明自主探索式开发,内部通过项目管理系统协调
  • 所以, gitlab 和 github 类似, 都是象校园一样的基础设施:
    • 不用学员自行安装, 是现成的网站, 报名后由课程方统一注册配置好各种权限以及辅助环境
  • 上课也和以往学校的形式根本性不同:
    • 除了定期直播, 要一起同一时刻进入同一虚拟空间来交流外
    • 其它所有时间(包含报名到开课的预习期间), 学员就已经可以通过 gitlab 开展自由讨论
    • 相互帮助, 快速根据具体问题来对应学习知识点
  • 而不是先完成"系列基础知识点"的所谓学习再来编程:
    • 编程, 只能通过编程来学会, 不可能通过其它形成
    • 所以, 从第一时间开始协作/编程, 是蟒营™课程最大的乐趣所在.
  • 即: 蟒营™的课程, 就在大家身边任何时刻开展:
    • 一但有时间, 打开电脑
    • 理解开发任务, 探索问题解决, 检验方案过程, 交流代码/经验....
    • 任何困难, 42分钟以内无法直觉解决时
    • 立即进入 gitlab 用 Issue 的预设模板来组织一篇正式问题
    • 大家就可以来针对足够的背景信息来针对性讨论和提供帮助
    • 而不用在 QQ/微信中无休止的低效率交流
    • ...
  • 所以,课程形式就是真实的开发:
    • 参与课程, 就是参与开发
    • 在开发过程中学习编程
    • 在编程过程中理解语言
    • 在交流过程中练习工程
    • 在工程环境中体验开发
    • ...

内容/效果¶

问: 你好 这个课程是线下的吗？

答: 是也乎,(￣▽￣)

  • 不是, 主要是网络课程, 看具体城市, 有助教的能聚集5人以上的,
    • 可以有定期线下答疑;
    • 北/上/广这类大城市, 一般都有助教,
    • 但是, 得看是否能定期有5人以上学员聚集起来了;
  • 要知道, 城市越大, 城市内交通成本也越大;
    • 比如说上海, 就算是在人民广场组织一次活动, 从闵行/浦东过来单程也要一个小时以上;
  • 所以, 网络课程好处就体现在突破物理空间约束;
    • 而且, 现在协作渠道, 也多为网络空间, 很难相象 OpenSSL 之类开源项目,
    • 能将分散在全球的开源聚集在一起, 每天开会...
  • 综上, 先享受在线交流和课程, 再用线下交流加强为好,
  • 反过来, 可以选择的就太少了...
  • 进一步的, 说句扎心的实话:
    • 之前大学4~6年, 可都是线下教学, 可学出来的有几成?

问: 网课 有时间限制吗?

答: 有的, 不同其它只提供资料, 付费后随便你学几年都好的,

  • 蟒营有严密期限:
    • 0: 因为以周任务为线索的学习, 必须保证其节奏
    • 1: 每周和同学们一起冲关, 比独自嗯哼要带感
    • 2: 课程期限以内, 助教时间是保证聚焦, 随时回答问题的
    • 3: 死线是最好的动力...
  • 当然, 蟒营又不是线下课程那种一毕业什么都没了...
    • 蟒营是自学型社区式课程,
    • 每期课程聚集起来的学习者, 天然变成共学社区,
    • 课程结束后, 依然有节奏不那么严密的社区可以继续学习,
  • 毕竟, 学习是终身的,
  • 但是, 学习方式和环境是珍稀的

    • 光是会相应工具/软件/开发语言,
    • 并不一定能组织/带领团队完成复杂的研究任务...
  • 这时的 能技/工程意识/团队气势/....
    • 才更加重要/关键,
    • 这些是其它编程类课程, 没法儿提供的了...
  • 101camp 系列课程, 则将这些设计为背景通识能力训练目标 ;-)

问: 按要求完成培训中的作业 那课程结束后能达到怎么样的效果 ?

答: 各种宣传中有说:

有能力组建5人以内开发团队, 亲手完成实用原型软件
  • 意思是:
    • 由你作为 leader
    • 可以合理组织 5 人以内团队,
    • 一起开发各种原型软件;
  • 所谓原型软件的意思:
    • 0: 非商用软件
    • 1: 可运行
    • 2: 可完成核心功能
    • 3: 解决最初目标问题
    • 4: 但是,无论界面/交互/文档/负荷/... 都不足以作为成品发售
  • 所谓实用:
    • 0: 解决真实问题
    • 1: 问题由你自行发现
    • 2: 输入的数据都是真实事件/来源/行为/...

转职目标:¶

答: 这真心是个严重的问题了;

  • 首先, 蟒营™系列课程筹备的都是入门班,解决的是越来越多想自学编程却无法得其门而入的基本问题;
    • 虽然原本就是蟒营原型 -> PythoniCamp 训练营的目的;
    • 但是, 后来发现, 行业变化太快, 每个岗位上要求的技能树也在飞速变化中,
    • 然后, 蟒营™课程就转向更加基础和不变的系列技能和习惯培养;
  • 所以, 决心转职变成职业软件工程师, 这是重大决定, 必要要改变的很多,
    • 在蟒营™入门班仅仅两个月的时间里, 很难全部完成;
  • 但是, 蟒营™课程可以完成最根本性的改变:
    • 从普通人, 变成有工程习惯的自学者;
    • 而软件工程师, 可以视为有高度自学自学效能的专业人士;
    • 那么, 通过蟒营™课程之后, 多久可以完成专职准备? 是否要上其它课?
  • 俺直觉的回答是:
    • 看你准备投递的职位是哪种, 如果是 BAT 之类大厂一线工程师, 目测还至少得3年积累;
    • 如果只是想先转换到软件行业, 那么, 根据兴趣或是目标岗位进行专门自学, 最多半年也足以胜任多数基础软件岗位了;
  • 只是, 无论哪种岗位, 都只期待 "有经验" 的相关工程师;
  • 所以, 想顺利入职, 首先得证明自己可以胜任;
  • 这也是蟒营™中反复强调的概念 -> MVP;
    • 即,必须通过一个个 MVP 作品来检验/引导/证明自己能力增长的路径;
    • 所以, 想入职前端工程师? 那至少应该先发布一个有良好移动端响应式统计的简历网站;
  • 想入职系统运维工程师? 那至少得先发布一个运行在 Linux 系统上的私人域名网站, 并包含一个 Wordpress 动态 blog, 自己针对涉及软件都真正了解, 并针对性优化过, 可以承载一定访问压力...是的压力是自行制造测试得到的精确数据;
  • 想入职移动应用开发? 那至少在主要市场中发布过自己的应用;
  • 想入职...任何职位前, 至少先有能力独立或是在技术社区中完成相应事务;
  • 那么, 以上各种实践目标什么课有涉及? ... 据俺观察, 可能很难找到.
  • 所以, 在蟒营™之后, 想快速完成职业化, 除了利用蟒营™中获得的自学经验外;
  • 可能最缺少的只是: 理论知识/实践经验;
    • 前者, 建议直接学习网络中公开的哈佛/MIT 之类名校公开课, CS101 系列的, 配合经典教程,足以完成弥补;
    • 后者, 要不参与一些技术社区的运维, 要不尝试一些公司实习生岗位;
  • 最好, 还是自行先尝试独立实现自己有兴趣的作品, 真正走一遍从构思到探索到尝试, 到重构, 到持续改进, 变成可用产品...这整个过程;
  • 没准, 无意间就直接完成了良好的创业项目, 被大厂收购了呢?
  • 嘦关注想解决的问题是否:
    • 0: 有用
    • 1: 有趣
    • 2: 有种
